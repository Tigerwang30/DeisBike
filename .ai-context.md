## Critical Env Vars

| Name | Purpose | Expires? |
|------|---------|---------|
| `JWT_SECRET` | Signs user JWTs | No |
| `NODE_ENV` | `development` enables dev-login, relaxed cookies | No |
| `LINKA_API_BASE_URL` | `https://app.linkalock.com/api/merchant_api` | No |
| `LINKA_ACCESS_TOKEN` | Meteor session token (X-Auth-Token header) | Yes — daily |
| `LINKA_USER_ID` | Meteor user ID (X-User-Id header) | Yes — daily |
| `STORAGE_BACKEND` | `json` (default) or `memory` | No |
| `STORE_PATH` | Path to JSON store file (default: `data/store.json`) | No |

LINKA_ACCESS_TOKEN and LINKA_USER_ID are NOT in bikes.json — they are fleet-level credentials in `.env`. Per-bike credentials (linka_token, mac_addr) live in `bikes.json`.

## Data Models

### Session (active ride, in store["sessions"])
```python
{
  "sessionId": "session-{timestamp}-{userId}",  # str, primary key
  "bikeId":    "5000",                           # str
  "userId":    "dev-user-001",                   # str
  "startTime": "2024-01-15T10:30:00",            # ISO UTC string
  "status":    "ride_active",                    # "ride_active" | "chain_unlocked"
  # Two-step flow only:
  "chainUnlocked": True,
  "wheelUnlocked": False,
}
```

### Ride Record (history, in store["ride_history"][userId][])
```python
{
  **session,          # all session fields copied
  "rideId":   "session-{timestamp}-{userId}",  # same as sessionId
  "endTime":  "2024-01-15T11:00:00",           # ISO UTC string
  "duration": 30,                              # int, minutes
  "status":   "completed",
}
```

### User (in store["users"][userId])
```python
{
  "id":              "dev-user-001",          # str
  "email":           "user@brandeis.edu",     # str
  "displayName":     "Dev User",              # str
  "photo":           None,                    # str | None
  "hasSignedWaiver": True,                    # bool
  "moodleApproved":  True,                    # bool
  "isAdmin":         False,                   # bool
  # Optional fields set after approval:
  "moodleApprovedAt": "2024-01-15T10:00:00",
  "moodleApprovedBy": "admin-user-id",
}
```

## API Endpoint Inventory

```
GET  /auth/dev-login              none         Creates dev user, sets cookie, redirects to /map
GET  /auth/me                     cookie       Returns user object (safe fields)
POST /auth/waiver                 cookie       Sets hasSignedWaiver=True, re-issues JWT
POST /auth/logout                 none         Deletes auth_token cookie
GET  /auth/status                 none         Returns { authenticated, user } — no auth required

GET  /api/bikes                   cookie       List all bikes from registry
GET  /api/bikes/locations/all     cookie       List bikes with lat/lng/name/location
GET  /api/bikes/{bike_id}         cookie       Single bike status (chainLocked, battery)

POST /api/command                 fully_approved  actions: open | lock | unlock_chain | unlock_wheel | status | active_session
POST /api/command/webhook         none            LINKA webhook; event: chain_locked

GET  /api/rides/active            cookie       Current user's active ride (or { active: false })
GET  /api/rides/history           cookie       Array of past ride records (newest first)
GET  /api/rides/{ride_id}         cookie       Single ride record

GET  /api/reports/ride/{id}/pdf   cookie       PDF receipt (streaming)
GET  /api/reports/history/pdf     cookie       Full history PDF (streaming)
GET  /api/reports/summary         cookie       { totalRides, totalDuration, averageDuration, firstRide, lastRide }

GET  /api/admin/users             admin        { users: [...] }
GET  /api/admin/pending-approvals admin        { pendingApprovals: [...] }
POST /api/admin/users/{id}/approve-moodle  admin  Sets moodleApproved=True
POST /api/admin/users/{id}/revoke-moodle   admin  Sets moodleApproved=False
POST /api/admin/users/{id}/grant-admin     admin  Sets isAdmin=True
GET  /api/admin/stats             admin        { totalUsers, approvedUsers, waiverSigned, totalRides }
```

Auth levels:
- `none` — no auth
- `cookie` — valid JWT cookie required (`get_current_user`)
- `fully_approved` — hasSignedWaiver AND moodleApproved (`get_fully_approved_user`)
- `admin` — isAdmin=True (`get_admin_user`)

## LINKA Integration Facts

- Base URL: `https://app.linkalock.com/api/merchant_api`
- All requests: POST (with body) or GET (status check)
- Required headers: `X-Auth-Token`, `X-User-Id`, `Origin: https://fleetview.linkalock.com`, `Referer: https://fleetview.linkalock.com/`
- Command body (per bike, from `registry.command_body(bike_id)`):
  ```json
  { "access_token": "<linka_token>", "mac_addr": "<mac>", "schedule": true, "firmware_version": "2.6.15", "smartkey_mac": "" }
  ```
- Endpoints used: `/command_unlock`, `/command_lock`, `/device_status/{bike_id}`
- Error codes: 200=queued (OK), 620=another command in progress (wait 5-10s), 504=expired credentials
- Execution: commands queue in LINKA cloud; execute when LINKA app on phone comes within Bluetooth range of lock
- `call_linka()` returns simulated success if `LINKA_API_KEY` env var is not set (dev mode)

## Store Interface Contract

```python
# All implemented by MemoryStore and JsonFileStore
get_session(session_id: str) -> dict | None
get_active_session_for_user(user_id: str) -> tuple[str, dict] | None
get_all_active_sessions() -> dict[str, dict]
put_session(session_id: str, data: dict) -> None
delete_session(session_id: str) -> None

get_ride_history(user_id: str) -> list[dict]      # newest first
append_ride(user_id: str, ride: dict) -> None      # inserts at index 0

get_user(user_id: str) -> dict | None
get_all_users() -> list[dict]
put_user(user_id: str, data: dict) -> None
update_user(user_id: str, updates: dict) -> None   # partial update, dict.update()
```

Access via `get_store()` from `api/storage/__init__.py` — never import `MemoryStore`/`JsonFileStore` directly.

## Business Rules

1. Users must `hasSignedWaiver=True` to access any ride endpoint
2. Users must `moodleApproved=True` to actually unlock a bike
3. Only one active ride per user (enforced by `get_active_session_for_user` returning the first match)
4. Bikes are identified by string ID (e.g. `"5000"`) — always string, never int
5. Session IDs format: `"session-{unix_ms}-{user_id}"`
6. Ride duration is computed server-side in minutes (integer); client displays seconds locally
7. `ride_history` stores records newest-first (`insert(0, ...)`)
8. Admin approval does not re-issue the user's JWT; user must log out/in to gain access

## Non-obvious Gotchas

1. **Phone required.** LINKA lock has no WiFi. Commands queue in cloud, execute only when LINKA app on phone is within Bluetooth range. If no phone is near, the lock command succeeds (API 200) but the lock does not move.

2. **LINKA credentials expire daily.** `LINKA_ACCESS_TOKEN` and `LINKA_USER_ID` are Meteor.js session tokens. They expire in ~24 hours. Rotate via FleetView DevTools → Network tab → copy x-auth-token and x-user-id from any POST request.

3. **Vercel kills in-process state.** Each Vercel function invocation may run in a fresh process. `JsonFileStore` writes to `/tmp` which is ephemeral on Vercel. Set `STORAGE_BACKEND=memory` on Vercel or use external persistence.

4. **`call_linka` simulates if no API key.** If `LINKA_API_KEY` is not set, `call_linka()` returns `{"success": True, "simulated": True}` without making any HTTP request. This enables frontend development without real credentials.

5. **JWT not updated on admin approval.** When an admin sets `moodleApproved=True`, this updates the store but not the user's JWT cookie. The user's next auth check still reads the old JWT. User must log out and log in again for the change to take effect. `get_fully_approved_user()` reads from the JWT, not the store.

6. **`bikes.json` is gitignored.** Contains per-bike LINKA tokens. Commit `bikes.example.json` instead. New developers copy it to `bikes.json` and fill in real values.

7. **Two-step unlock flow is lightly used.** `unlock_chain` + `unlock_wheel` are in `command.py` as thin inline logic (not extracted to `RideService`). The `open` action (single-step) is preferred and used by the current UI. The two-step flow exists for future hardware that distinguishes chain from wheel locking.

8. **`MapPage` uses `bikeService.getAll()`** (returns name, location, available) not `getLocations()` (returns lat/lng only). Both endpoints exist; `getAll` is the right one for the card grid.

9. **`RideContext` only polls when `user.moodleApproved`.** If a user is logged in but not approved, the polling hook is disabled (`enabled=false`) and `useRide()` returns `{ active: false, loading: false }`.

10. **`data/store.json` parent dir must exist.** `JsonFileStore._save()` calls `os.makedirs(os.path.dirname(path), exist_ok=True)`. The `data/` directory is gitignored and created automatically on first write.

## Import Conventions

- Backend routes import services via module singletons: `from api.services.ride_service import ride_service`
- Routes never import `_state.py` (deleted) — always use `get_store()`
- Routes never import `_command_body()` from `_linka.py` — always use `registry.command_body(bike_id)`
- Frontend pages import from specific service files: `import { commandService } from '../services/commands'`
- Frontend pages never import from `'../services/api'` — that monolith is superseded by per-domain files
